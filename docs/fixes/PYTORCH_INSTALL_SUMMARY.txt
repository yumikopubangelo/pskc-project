================================================================================
PyTorch Installation Issue - Fixed
================================================================================

PROBLEM
-------
When running the Docker container, warning appears:
  "PyTorch not available, LSTM model disabled"

This occurs even though PyTorch should be installed.

ROOT CAUSE
----------
Docker multi-stage build issue:

1. Builder stage: PyTorch installed WITHOUT --prefix=/install
   → Went to default Python site-packages in builder
   
2. Runtime stage: Only copied /install directory from builder
   → PyTorch wasn't in /install, so it didn't transfer
   → Result: TORCH_AVAILABLE = False in src/ml/model.py

SOLUTION
--------
✅ Changed Dockerfile:

BUILDER STAGE (line 29):
  OLD: RUN pip install --no-cache-dir torch==2.3.1+cpu ...
  NEW: RUN pip install --prefix=/install --no-cache-dir torch==2.3.1+cpu ...

RUNTIME STAGE (line 60):
  Added: RUN pip install --no-cache-dir torch==2.3.1+cpu ... (as backup)

This ensures:
1. PyTorch in builder goes to /install (gets copied to runtime)
2. PyTorch also installed in runtime as redundancy

FILES MODIFIED
--------------
- Dockerfile (lines 27-35 in builder, lines 50-62 in runtime)

DEPLOYMENT
----------
1. Rebuild Docker:
   cd d:\pskc-project
